# 智慧云脑诊疗平台 - 核心业务流程主干与跨流交互规范

本文档提炼自“智慧云脑诊疗平台业务模块图”，聚焦于系统最核心的业务主干流程以及各流程线之间的关键跨界交互，旨在为系统开发与边界设计提供精炼的全局视图。

---

## 1. 四大核心业务流程主干

### 1.1 患者入流线（蓝线：诊前准备）
患者从触达到获取号源的标准化路径，核心在于通过 AI 实现前端分流。
* **流程路径：** `患者端` ➔ `注册/登录` ➔ `AI智能问诊 (基于AI分诊台进行病情初步评估与科室引导)` ➔ `线上挂号 (进入对应科室与医生的候诊池)`

### 1.2 医生门诊线（绿线：诊中枢纽）
全系统的核心业务枢纽，汇聚患者流量并向下游业务流转指令。
* **流程路径：** `医生端` ➔ `登录` ➔ `门诊工作台 (融合“线上挂号”与“线下挂号”队列)` ➔ `诊疗互动 (读写电子病历 / AI助理医生实时伴诊提示)` ➔ `开具医嘱 (分发至检查检验或药房管理)` ➔ `结束就诊`

### 1.3 智能化医技线（红线：异步算力）
处理高密度医学影像计算与大模型推理的高性能闭环链路。
* **流程路径：** `门诊下达影像检查` ➔ `执行检查/检验 (如头部CT)` ➔ `头部CT医学影像识别 (后台AI模型计算)` ➔ `生成AI智能检查报告` ➔ `结果回传至检查检验模块`

### 1.4 运筹调度线（橙线：后勤基座）
系统的管理与资源供给基础，决定前台业务的吞吐上限。
* **流程路径：** `管理员端` ➔ `登录` ➔ `AI医生排班系统` ➔ `释放/更新全院号源与出诊计划`

## 2. 认证授权安全层（横切关注点）

四大业务流程的入口均受统一认证授权体系管控，确保医疗数据与处方的访问合规。

* **认证入口：** 各端（患者端、医生端、管理员端）登录时，统一经 API Gateway 转发至认证服务，获取 JWT 访问令牌与刷新令牌。
* **角色模型（RBAC）：**
  * `Patient` — 患者：仅可访问本人病历、挂号记录
  * `Doctor` — 医生：可访问所属科室的接诊患者病历，开具医嘱与处方
  * `Technician` — 医技人员：可上传影像/检验数据，查阅关联检查申请单
  * `Admin` — 管理员：管理排班、号源与系统配置
* **令牌传播：** 微服务间 RPC 调用通过 OpenFeign 拦截器自动携带 JWT；Kafka 事件消息体中内嵌 `principal` 字段（操作人 ID + 角色），消费者据此进行操作审计与数据级权限校验。
* **处方/医嘱的强授权：** 开具处方属于高风险操作，需在 JWT 之外叠加二级确认（如二次密码验证或动态令牌），防止 session 劫持导致的处方滥用。

## 3. 核心跨流程交互契约

系统的高效运转依赖于以下四个关键的跨流程交互节点，这些节点实现了数据流与控制流的闭环。

```
[管理员端：AI排班] ────释放号源────➔ [患者端：线上/线下挂号]
                                              │
                                           队列合并
                                              ▼
[医技/AI：影像识别] ◄───下达检查医嘱─── [医生端：门诊工作台] ◄───伴诊监听─── [AI助理医生]
         │                                    ▲
         └───────────智能报告回传──────────────┘
```

1. **调度 ➔ 挂号 ➔ 门诊（资源流转交互）**
   * 管理员端 `AI医生排班` 动态生成的出诊计划，直接解锁并初始化 `患者端` 的号源池。
   * 患者端的 `线上挂号` 与医院现场的 `线下挂号` 产生的事件流，在 `门诊` 节点进行多源队列合并，形成医生的唯一工作队列。

2. **门诊 ➔ 医技 ➔ AI ➔ 门诊（影像检查闭环）**
   * 医生在 `门诊` 开具检查医嘱，门诊工作台通过 **OpenFeign 同步调用**医嘱路由服务，路由服务持久化医嘱单后立即返回 `accepted` 状态；若同步调用超时或失败，医生端即时感知并重试。
   * 医嘱被接受后，路由服务异步发布 `OrderIssuedEvent` 激活下游 `检查/检验` 流程。
   * 检查设备产生影像后，异步触发 `头部CT医学影像识别`，AI 计算完成后将 `AI智能检查报告` 与原始影像绑定，回传至医生 `门诊` 工作台亮起红点/弹窗提示。

3. **门诊 ⮂ AI助理（诊中人机协同）**
   * 医生进入 `门诊` 诊疗状态时，`AI助理医生` 旁路启动。医生端通过 **WebSocket 长连接**将实时语音流传至 AI 助理服务（经 ASR 转写为文本），AI 助理动态向医生工作台推送鉴别诊断与用药合规提醒。
   * 就诊结束后，AI 助理将本次会话的结构化摘要通过 `ConsultationAiSummaryEvent` 异步写入病历归档。

4. **门诊 ➔ 药房（诊后业务延伸）**
   * 医生在 `门诊` 确认处方并点击”结束就诊”，门诊工作台 **同步调用**医嘱路由服务确认处方已被接收（`accepted`），路由服务随后异步发布事件流向 `药房管理` 模块，触发配药与库存扣减。
