## 4. 数据库

每个微服务使用独立的 MySQL schema，通过 Nacos + OpenFeign 通信，不共享数据库。

| 微服务 | Schema | 核心表 |
|---|---|---|
| cloud-brain-auth | `cloud_brain_auth` | `user`, `role`, `refresh_token` |
| cloud-brain-outpatient | `cloud_brain_outpatient` | `consultation`, `emr`, `outpatient_outbox` |
| cloud-brain-imaging | `cloud_brain_imaging` | `imaging_task`, `imaging_report`, `imaging_outbox` |
| cloud-brain-scheduling | `cloud_brain_scheduling` | `schedule`, `queue`, `scheduling_outbox` |
| cloud-brain-order-routing | `cloud_brain_order_routing` | `medical_order`, `order_outbox` |
| cloud-brain-patient | `cloud_brain_patient` | 预留 |
| cloud-brain-admin | `cloud_brain_admin` | 预留 |

> **连接参数：** `jdbc:mysql://mysql:3306/{schema}?useUnicode=true&characterEncoding=utf8mb4`  
> **凭据：** `root / ${MYSQL_PASSWORD}`（仅开发环境）

## 7. 通用配置项

所有服务共享以下配置结构（`application.yml`）：

```yaml
server.port                        # 按服务分配 8081~8088
spring.application.name            # cloud-brain-{service}
spring.cloud.nacos.discovery.server-addr: nacos:8848
spring.kafka.bootstrap-servers: kafka:9092

management.endpoints.web.exposure.include: health
management.tracing.sampling.probability: 1.0
```

**数据库服务** 额外配置：
```yaml
spring.datasource.url               # jdbc:mysql://mysql:3306/{schema}
spring.datasource.username: root
spring.datasource.password: ${MYSQL_PASSWORD}
spring.jpa.hibernate.ddl-auto: validate
```

**Outpatient 专属**：
```yaml
spring.elasticsearch.uris: http://elasticsearch:9200
spring.cloud.openfeign.client.config.order-routing.connect-timeout: 3000
spring.cloud.openfeign.client.config.order-routing.read-timeout: 5000
```

**Auth 专属**：
```yaml
jwt.secret: ${JWT_SECRET}             # 生产环境通过环境变量注入
jwt.access-expiration: 900            # 15 min
jwt.refresh-expiration: 604800        # 7 days
```

## 8. Outbox 轮询参数

| 配置项 | 推荐值 | 说明 |
|---|---|---|
| 轮询间隔 | 200ms ~ 1s | `@Scheduled(fixedDelay = 500)` |
| 批量大小 | 100 条/次 | 每次从 outbox 表取未发送事件上限 |
| 投递重试 | 3 次，指数退避 | 失败后标记 `retry_count + 1` |
| 清理策略 | 已投递事件 7 天后删除 | 定时清理，防止 outbox 表膨胀 |

## 9. Feign 超时配置（医嘱同步调用）

| 参数 | 值 | 说明 |
|---|---|---|
| connect-timeout | 3000ms | 连接超时 |
| read-timeout | 5000ms | 读取超时 |
| 重试次数 | 0（不自动重试） | 由前端手动重试，保证幂等 |
