## 2. Auth — 认证授权

### 2.1 登录

```
POST /auth/login
Content-Type: application/json
```

**请求：**
```json
{
  "username": "doctor1",
  "password": "test123"
}
```

**响应 200：**
```json
{
  "accessToken":  "eyJhbGciOiJIUzI1NiJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiJ9...",
  "userId":       "9434c018-6e2d-403b-97c6-5d13131f238a",
  "name":         "张医生",
  "role":         "DOCTOR"
}
```

| 字段 | 说明 |
|---|---|
| `accessToken` | JWT，15 分钟有效 |
| `refreshToken` | 刷新令牌，7 天有效 |
| `role` | `PATIENT` / `DOCTOR` / `TECHNICIAN` / `ADMIN` |

**错误：** 403（凭据无效）

### 2.2 刷新令牌

```
POST /auth/refresh
X-Refresh-Token: <refreshToken>
```

**响应 200：** 同登录返回新 `accessToken`

### 2.3 二次密码验证（处方强授权）

```
POST /auth/verify-credential
Content-Type: application/json
```

**请求：**
```json
{
  "userId": "9434c018-...",
  "password": "test123"
}
```

**响应：** 200 通过 / 403 拒绝

## 4. Outpatient — 门诊工作台（核心）

### 4.1 开始接诊

```
POST /api/consultations/start
```

**请求：**
```json
{
  "consultationId": "c-001",
  "doctorId": "d-001"
}
```

**状态流转：** `WAITING → IN_PROGRESS`

**副作用：** 写入 `outpatient_outbox` → Kafka `outpatient-consultation-events`

**错误：** 500（已接诊或已结束的 Consultation 不可重复开始）

### 4.2 结束就诊

```
POST /api/consultations/{id}/finish
```

**状态流转：** `IN_PROGRESS → FINISHED`

**副作用：** 写入 `outpatient_outbox` → Kafka

### 4.3 写病历

```
POST /api/emr
```

**请求：**
```json
{
  "consultationId": "c-001",
  "content": "{\"chiefComplaint\":\"头痛伴恶心\",\"presentIllness\":\"额部搏动性头痛\"}"
}
```

**响应：**
```json
{
  "id": "fa93ec92-...",
  "consultationId": "c-001",
  "content": "{\"chiefComplaint\":\"头痛伴恶心\",...}",
  "createdAt": "2026-06-12T10:59:27.169015Z",
  "updatedAt": "2026-06-12T10:59:27.169015Z"
}
```

**副作用：** 异步索引到 ES `emr` 索引

### 4.4 更新病历

```
PUT /api/emr/{id}
Content-Type: application/json
```

**请求：**
```json
{ "content": "{\"chiefComplaint\":\"更新后的主诉\",...}" }
```

### 4.5 搜索病历（全文检索）

```
GET /api/emr/search?keyword=头痛
```

**响应：**
```json
[
  {
    "id": "ccfa0e8d-...",
    "chiefComplaint": "头痛伴恶心",
    "presentIllness": "3天前额部搏动性头痛",
    "consultationId": "c-001",
    "doctorId": "d-001",
    "patientId": "p-001"
  }
]
```

**说明：** 搜索 `chiefComplaint` + `presentIllness` 两字段，ES multi_match 查询

### 4.6 按患者查病历

```
GET /api/emr/patient/{patientId}
```

## 5. OrderRouting — 医嘱执行

### 5.1 提交医嘱

```
POST /api/orders
```

**请求：**
```json
{
  "idempotencyKey": "unique-key-123",
  "consultationId": "c-001",
  "orderType": "IMAGING",
  "orderDetails": "{\"type\":\"CT\",\"bodyPart\":\"head\"}"
}
```

| `orderType` | 含义 |
|---|---|
| `IMAGING` | 影像检查 |
| `LAB` | 检验 |
| `PRESCRIPTION` | 处方 |

**`idempotencyKey` 规则：** `consultationId + "_" + MD5(content)` 确保同一次就诊的相同内容医嘱不会重复创建

**响应 200：**
```json
{
  "orderId": "ce85276c-6da5-4c0e-8253-19e6dba58b94",
  "status": "ACCEPTED"
}
```

**状态流转：** `PENDING → ACCEPTED`（同步返回） → Outbox → Kafka `medical-order-events` → 下游异步执行

**幂等：** 重复提交相同 `idempotencyKey` 返回原 `orderId` 和状态，不会创建新记录

## 6. Imaging — 医学影像

### 6.1 提交影像

```
POST /api/imaging/submit
```

**请求：**
```json
{
  "consultationId": "c-001",
  "orderId": "o-001",
  "modality": "CT",
  "bodyPart": "HEAD",
  "filePath": "/data/dicom/ct001.dcm",
  "fileMd5": "a1b2c3d4e5f6"
}
```

| `modality` | 含义 |
|---|---|
| `CT` | 计算机断层扫描 |
| `MRI` | 磁共振 |
| `XRAY` | X 线 |

**响应：**
```json
{
  "id": "0056d341-...",
  "consultationId": "c-001",
  "status": "COMPLETED",
  "modality": "CT",
  "bodyPart": "HEAD",
  "createdAt": "2026-06-11T21:25:25.567902Z"
}
```

**处理流程：** `RECEIVED → PREPROCESSING → INFERRING → COMPLETED`，完成后发布 `CtReportGeneratedEvent` 到 Kafka

### 6.2 查看任务状态

```
GET /api/imaging/tasks/{id}
```

状态值：`RECEIVED` / `PREPROCESSING` / `INFERRING` / `COMPLETED` / `FAILED`

### 6.3 查看 AI 报告

```
GET /api/imaging/reports/{taskId}
```

**响应：**
```json
{
  "id": "report-001",
  "imagingTaskId": "0056d341-...",
  "findings": "{\"nodules\":[...]}",
  "measurements": "{\"nodule_01\":{\"diameter_mm\":8,...}}",
  "impression": "右肺上叶磨玻璃结节，建议 6 个月随访复查",
  "criticalWarning": false
}
```

## 7. Scheduling — 排班与队列

### 7.1 排班 CRUD

```
POST   /api/scheduling/schedules            创建排班
GET    /api/scheduling/schedules/{doctorId}  查询排班（?date=2026-06-12）
```

**创建请求：**
```json
{
  "doctorId": "d-001",
  "doctorName": "张医生",
  "departmentId": "DEPT-001",
  "date": "2026-06-12",
  "startTime": "08:00",
  "endTime": "12:00",
  "maxPatients": 20
}
```

### 7.2 队列管理

```
POST   /api/scheduling/queue                  患者加入队列
POST   /api/scheduling/queue/{id}/check-in    签到
POST   /api/scheduling/queue/call-next        叫号（?doctorId=xxx）
POST   /api/scheduling/queue/{id}/complete    完成就诊
POST   /api/scheduling/queue/{id}/pass        过号
POST   /api/scheduling/queue/{id}/cancel      取消
GET    /api/scheduling/queue/{doctorId}       查看队列
```

**加入队列请求：**
```json
{
  "doctorId": "d-001",
  "patientId": "p-001",
  "patientName": "李患者",
  "source": "ONLINE"
}
```

**状态机：**
```
WAITING →(签到)→ IN_QUEUE →(叫号)→ IN_CONSULTATION →(完成)→ COMPLETED
                                                    →(过号)→ PASSED
         →(取消)→ CANCELLED
```

**队列变更事件：** 每次状态变更 → `scheduling_outbox` → Kafka `doctor-scheduling-events`

## 9. AI Assistant — WebSocket 实时通信

### 连接

```
ws://localhost:8080/ws/ai-assistant    （通过 Gateway）
ws://localhost:8084/ws/ai-assistant   （直连）
```

协议：STOMP over WebSocket

### 9.1 启动会话

```
SEND /app/start-session
```

**请求：**
```json
{
  "consultationId": "c-001",
  "doctorId": "d-001"
}
```

**响应（推送到 `/topic/session`）：**
```json
{
  "id": "session-uuid",
  "consultationId": "c-001",
  "doctorId": "d-001",
  "active": true
}
```

### 9.2 发送问诊文本

```
SEND /app/transcript
```

**请求：**
```json
{
  "sessionId": "session-uuid",
  "text": "患者说头痛三天"
}
```

**响应（推送到 `/topic/suggestion/{sessionId}`）：**
```json
{
  "text": "考虑偏头痛或紧张性头痛，建议询问发作频率、持续时间及伴随症状"
}
```

### 9.3 结束会话

```
SEND /app/end-session
```

**请求：**
```json
{ "sessionId": "session-uuid" }
```

**副作用：** 发布 `ConsultationAiSummaryEvent` 到 Kafka `ai-assistant-events`

## 10. Kafka 事件（前端可感知的变化）

虽然是异步，但前端需要关注以下事件触发的 UI 变化：

| Topic | 事件 | 前端反应 |
|---|---|---|
| `medical-imaging-events` | `CtReportGeneratedEvent` | 门诊工作台红点/弹窗"CT 报告已生成" |
| `medical-order-events` | `OrderFulfilledEvent` | 医嘱状态更新为"已完成" |
| `medical-order-events` | `OrderRejectedEvent` | 弹窗告警"医嘱被驳回" |
| `doctor-scheduling-events` | `QueueUpdatedEvent` | 刷新候诊列表 |
| `ai-assistant-events` | `ConsultationAiSummaryEvent` | 病历摘要自动归档 |

> 当前 WordPress 未实现实时推送，前端可通过**轮询对应 REST API** 模拟（1-3 秒间隔）。

## 12. 错误响应格式

所有 500 错误统一格式：

```json
{
  "timestamp": "2026-06-12T10:51:46.163+00:00",
  "status": 500,
  "error": "Internal Server Error",
  "path": "/api/emr"
}
```

503 表示服务不可达（需检查 Nacos 注册状态）。

## 13. 前端轮询建议

目前 WebSocket 推送仅 AI Assistant 已实现。其余异步通知可用轮询替代：

| 需要感知的变化 | 轮询接口 | 建议间隔 |
|---|---|---|
| 候诊列表变更 | `GET /api/scheduling/queue/{doctorId}` | 2s |
| 医嘱状态 | 扩展 `GET /api/orders/{id}` 接口 | 3s |
| CT 报告生成 | `GET /api/imaging/tasks/{id}` | 3s |
| 病历搜索 | `GET /api/emr/search?keyword=` | 用户触发的单次请求 |
