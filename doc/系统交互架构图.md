## 2. 服务与 MySQL 的读写关系

```
                    ┌─────────────────────────────┐
                    │         MySQL :3306          │
                    └─────────────────────────────┘
          ┌──────────┬──────────┬──────────┬──────────┐
          │          │          │          │          │
    ┌─────▼──┐ ┌────▼───┐ ┌───▼────┐ ┌───▼────┐     │
    │  Auth  │ │Outpat. │ │Imaging │ │Schedul.│  ... │
    └───┬────┘ └──┬──┬──┘ └───┬────┘ └───┬────┘     │
        │         │  │         │          │          │
        ▼         ▼  ▼         ▼          ▼          ▼
    ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
    │user  │ │consul│ │image │ │sched │ │order │  ...
    │role  │ │emr   │ │report│ │queue │ │      │
    │token │ │outbox│ │outbox│ │      │ │outbox│
    └──────┘ └──────┘ └──────┘ └──────┘ └──────┘

    每个服务只读写自己的 Schema，互不跨库。
    Outbox 表在各自 Schema 内，与业务表同事务写入。
```

## 4. 医嘱两阶段交互

```
     Outpatient                        OrderRouting
         │                                  │
         │  POST /api/orders (同步 RPC)      │
         │  timeout: connect 3s, read 5s     │
         ├─────────────────────────────────►│
         │                                  │  ┌── MySQL 事务 ──┐
         │                                  ├──┤ INSERT order   │
         │                                  │  │ INSERT outbox  │
         │                                  │  └────────────────┘
         │       200 {orderId, ACCEPTED}     │
         ◄─────────────────────────────────┤
         │                                  │
    医生看到 ✔                              │
         │                                  │  (异步阶段)
         │                                  │  Outbox → Kafka
         │                                  ├────► OrderIssuedEvent
         │                                  │         │
         │                                  │    ┌────▼────┐
         │                                  │    │  药房    │
         │                                  │    │ 检查科室 │
         │                                  │    └─────────┘
```

## 图例

| 线条 | 含义 |
|---|---|
| `──►` 实线箭头 | 同步 RPC 调用（HTTP/Feign） |
| `--►` 虚线箭头 | 异步消息（Kafka） |
| `~~►` 波浪线 | WebSocket 实时流 |
| `···` 点线 | 依赖引用（Maven） |
| `★` | 核心域 |
